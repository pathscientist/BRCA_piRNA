# Advanced downstream analyses: required files and columns

This note summarizes what you need **in addition to** the 7 processed piRNA datasets:

- `processed_results/BRCA1_processed.csv`
- `processed_results/PRJNA294226_processed.csv`
- `processed_results/PRJNA482141_processed.csv`
- `processed_results/PRJNA808405_processed.csv`
- `processed_results/PRJNA934049_processed.csv`
- `processed_results/yyfbatch1_processed.csv`
- `processed_results/yyfbatch2_processed.csv`

## 1) Additional external input files (optional)

### A. Label mapping file (only if `label` is missing in processed CSV files)
- File: any CSV (e.g., `processed_results/labels_mapping.csv`)
- Required columns:
  - `sample_id`
  - `label` (`Normal`/`Cancer`)
- Config:
  - set `config$input$labels_path` to this file.

If `label` already exists in each processed file, you do **not** need this extra file.

### B. Drug sensitivity reference data (optional, for real drug prediction)
- The current pipeline provides a placeholder output (`drug_sensitivity_note.txt`) for this step.
- To run actual prediction, add your preferred reference resource/tool inputs for `oncoPredict`/`pRRophetic` workflows.

## 2) Required columns inside processed CSV files (for advanced downstream)

The base diagnostic pipeline needs expression + labels. Advanced modules require extra columns:

- Core fields:
  - `sample_id`
  - `label`
  - piRNA expression columns
- Strongly recommended metadata:
  - `batch`
  - `cohort`
- For subgroup plots:
  - `age`
  - `stage`
  - `subtype`
  - (optional) sex/smoking/histology if you want those panels
- For survival analyses:
  - `OS_time`
  - `OS_event` (0/1)
- For TMB association:
  - `TMB` (optional; only needed for TMB analyses)

## 3) Generated prerequisite artifacts (must exist before advanced modules)

These are not user-supplied files, but they must be generated by prior scripts:

1. From model-selection stage:
- `artifacts/models/best_model_artifact.rds`

2. From subgroup analysis stage:
- `artifacts/reports/merged_subgroup_scored_data.rds`

3. For gene-network module (from downstream correlation stage):
- `artifacts/reports/pearson_correlated_genes.csv`

## 4) Script dependencies by module

### `pipeline/07_clinical_subgroup_analysis.R`
Needs:
- merged/scored data columns for subgroup and score analyses (`age`/`stage`/`subtype`, labels)
- survival columns for Cox (`OS_time`, `OS_event`)

### `pipeline/08_downstream_biology_survival_meta.R`
Needs:
- feature/correlation-compatible expression table
- optional enrichment/network/meta packages (`ReactomePA`, `clusterProfiler`, `ggraph`, `igraph`, `survminer`, `pheatmap`, `metafor`)

### `pipeline/09_gene_network_analysis.R`
Needs:
- `artifacts/reports/pearson_correlated_genes.csv` with columns:
  - `piRNA`, `gene`, `cor`

### `pipeline/10_advanced_clinical_omics_analysis.R`
Needs:
- `artifacts/models/best_model_artifact.rds`
- `artifacts/reports/merged_subgroup_scored_data.rds`
- survival columns (`OS_time`, `OS_event`)
- optional `TMB`
- optional packages for extended plots/models (`timeROC`, `rms`, `survminer`, `immunedeconv`)

## 5) Minimal practical checklist

If you want to run all advanced analyses smoothly, prepare:

1. The 7 processed CSV files (already listed).
2. Ensure each file includes at least:
   - `sample_id`, `label`, expression columns, `OS_time`, `OS_event`, `age`, `stage`, `subtype`, `batch`, `cohort`.
3. Optional: add `TMB` for TMB tests.
4. Optional: add `labels_mapping.csv` only when `label` is missing in source files.
5. Run scripts in order to generate prerequisite artifacts before 08/09/10.

## 6) Best independent cohort selection (yyfbatch1 vs yyfbatch2)

Use `pipeline/11_select_best_independent_cohort.R` to train/evaluate both choices and pick the best external cohort automatically.

It enforces:
- external AUC >= 0.80
- feature count <= 10

Outputs:
- `artifacts/batch_compare_summary.csv`
- `artifacts/batch_compare_best.csv`
- `artifacts/batch_compare_summary.md`
